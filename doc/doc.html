Processamento de Linguagem Natural, tambm conhecida pelo acrnimo do termo em ingls, NLP, uma subrea da cincia da computao e da inteligncia artificial que se preocupa com a interao de computadores e contedo lingustico produzido por humanos, seja ele falado ou escrito.

Apesar de estudos em NLP remontarem h pelo menos 50 anos, recentemente, com a revoluo do aprendizado profundo na rea de aprendizado de mquina, houveram muitas inovaes extremamente importantes, como a criao de redes neurais recursivas e modelos vetoriais de representao de palavras, como Word2Vec [1] e GloVe.

Modelagem e Implementao

Bases de Dados

Como dito na seo de introduo, o corpus utilizado foi retirado de http://mattmahoney.net/dc/text8.zip.

Os dados utilizados para validao foram retirados do documento question-words.txt presente no repositrio do Word2Vec do Google [2]. Este arquivo contm uma srie de perguntas de analogia, separadas por categorias, e no formato "Se A est para B, C est para D", em cima das quais o modelo gerado tem que prever a palavra D ao subtrair o vetor de A do vetor de B e somar o resultado ao vetor de C.

Implementao

A implementao mais conhecida do Word2Vec em C e feita pelo time do Google Brain [2]. No entanto, escolhemos implementar o trabalho todo em Python3, que, alm de ser uma linguagem de alto nvel e fcil de utilizar, possui a biblioteca gensim, pronta para lidar com a maioria das tarefas de NLP de forma simples e eficiente.

O cdigo como um todo bem simples. As formas de avaliao dos modelos usadas foram a acurcia mdia, obtida com a funo evaluate_word_analogies() da biblioteca gensim e distncia do cosseno mdia entre as palavras preditas e as esperadas, implementada na funo evaluate_analogies_distance(). Em ambas as avaliaes, palavras fora do vocabulrios foram desconsideradas.

O corpus usado j est totalmente tratado, ento s foi necessria uma funo - split_corpus_file() - para que ele fosse quebrado nos tamanhos necessrios para experimentao. Essa funo s precisa ser executada uma vez, j que os corpus de tamanhos diferentes so salvos em arquivos separados. O nico tratamento textual necessrio neste trabalho prtico, feito na funo prepare_validation_file(), foi o do conjunto de dados de validao, o qual precisa de ter suas categorias retiradas e suas palavras transformadas para lowercase, para evitar conflitos no clculo da distncia.

Execuo do Cdigo

Para treinar os modelos, devemos chamar o script train.py. Os parmetros a serem variados j esto descritos no script, ento s precisamos de passar dois argumentos na chamada como a seguir:

Teste

Para avaliar os modelos treinados, devemos chamar o script test.py. Os testes so executados sequencialmente em todos os arquivos de modelos e salvos na pasta de resultados passada como um dos argumentos na chamada:

python3 test.py /path/to/validation/file /path/to/models/dir/ /path/to/results/dir/

Anlise Experimental

Parmetros

Os parmetros variados foram:

  • Tamanho do corpus: 25%, 50%, 75% e 100% do corpus original.
  • Tamanho do contexto: 5, 10, 20 e 100 palavras.
  • Arquitetura: CBOW ou Skip-Gram.

Ao realizarmos todas as combinaes possveis de parmetros acabamos com 32 combinaes nicas.

Avaliao de Modelos

Nossos modelos foram avaliados em termos da acurcia mdia nos dados de validao e da distncia do cosseno mdia entre as palavras preditas para cada pergunta e as palavras esperadas (respostas corretas).

import os
import pandas as pd

pd.set_option("display.max_rows", 64)  # Total number of rows

# Data dictionary to convert to pandas DataFrame
data = {
        'corpus_proportion': [],
        'context_size': [],
        'measure_type': [],
        'measure_value': []
}

# Execution parameters
corpus_proportions = ('0.25', '0.5', '0.75', '1')
context_sizes = ('5', '10', '20', '100')
training_algorithms = {'cbow': 'CBOW', 'sg': 'Skip-Gram'}

# Read files and populate data dictionary
for cp in corpus_proportions:
    for cs in context_sizes:
        for ta in training_algorithms.keys():
            
            with open('../results/{}-{}-{}.txt'.format(cp, cs, ta), 'r') as f:
                for line in f.readlines():
                    if line.startswith('accuracy'):
                        accuracy = line.split('=')[1][:-1]  # [:-1] to remove trailing '\n'
                        
                        data['corpus_proportion'].append(cp)
                        data['context_size'].append(cs)
                        data['measure_type'].append(training_algorithms[ta] + ' accuracy')
                        data['measure_value'].append(float(accuracy))
                    elif line.startswith('distance'):
                        distance = line.split('=')[1]
                        
                        data['corpus_proportion'].append(cp)
                        data['context_size'].append(cs)
                        data['measure_type'].append(training_algorithms[ta] + ' distance')
                        data['measure_value'].append(float(distance))

# Create pandas DataFrame and order columns
df = pd.DataFrame(data)[['corpus_proportion', 'context_size', 'measure_type', 'measure_value']]

# Force context_size to be ordered for a correct plot with facet_wrap
df['context_size'] = pd.Categorical(df['context_size'], categories=['5', '10', '20', '100'], ordered=True)

import plotnine as pn

FONT = 'Roboto'
FONT_SIZE = 12
TEXT_COLOR = '#767676'

def theme_nlp():
    return (pn.theme_bw(base_size=FONT_SIZE, base_family=FONT)
            + pn.theme(axis_text_x = pn.element_text(color=TEXT_COLOR),
                       strip_background = pn.element_blank(),
                       legend_key = pn.element_blank(),
                       legend_title = pn.element_text(size=FONT_SIZE, margin={'b': 14}, weight='bold')
                      )
            )

Anlise

O grfico acima est agrupado por tamanhos de contexto. Com a exceo da distncia mdia quando temos um contexto de tamanho 10 e 75% do corpus, a arquitetura Skip-Gram predominou em todos os resultados, para ambas as mtricas de avaliao. Buscamos idealmente uma maior acurcia e uma menor distncia.

Apesar de a mtrica de distncia do cosseno mdia piorar a medida que o corpus aumenta, j que agora temos mais palavras no corretas que podemos predizer, o comportamento da acurcia mdia no to facilmente sumarizado. Em geral temos os piores valores para corpus pequenos, j que o modelo no consegue aprender de forma satisfatria. Essa acurcia tende a crescer para corpus de 50% e 75% do original, porm acaba sendo pior na maioria dos casos quando usamos o corpus completo.

interessante notar que os autores do algoritmo Word2Vec recomendam um tamanho de contexto igual a 5 para a arquitetura CBOW e 10 para Skip-Gram [2]. Perceba que os valores so pequenos, e que a medida que aumentamos o contexto a distncia mdia das predies vai piorando, exceto para o contexto de tamanho 100, no qual, incrivelmente temos melhores resultados do que para casos anteriores (com exceo do tamanho 5). Esse comportamento de certa forma contra-intuitivo e difcil de explicar.

Os melhores resultados tanto para acurcia mdia quanto para distncia mdia foram alcanados com um contexto de tamanho 5, 25% do corpus e arquitetura Skip-Gram.

Concluso

Com este trabalho buscamos aprender um pouco a respeito da construo de modelos vetoriais para representao de palavras ao usarmos o algoritmo Word2Vec com as arquiteturas CBOW e Skip-Gram para construo dos modelos. Ao todo foram construdos 32 modelos, cada um com uma combinao diferente de parmetros para o tamanho do corpus, o tamanho do contexto e a arquitetura utilizada.

A arquitetura que predominou em quase todos os casos foi a Skip-Gram. Com a exceo da distncia mdia quando temos um contexto de tamanho 10 e 75% do corpus, a arquitetura Skip-Gram predominou em todos os resultados, para ambas as mtricas de avaliao. Os melhores resultados tanto para acurcia mdia quanto para distncia mdia foram alcanados com um contexto de tamanho 5, 25% do corpus e arquitetura Skip-Gram.

Uma futura comparao do tempo de execuo dos modelos com as ambas as arquiteturas pode nos mostrar se h um trade-off entre qualidade e custo computacional, para nos ajudar a escolher um modelo.
